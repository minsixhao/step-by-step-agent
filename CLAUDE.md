# Step-by-Step Agent 项目说明

## 项目概述

这是一个从 Demo 一步步演进到产品级的 Agent 学习项目。采用"目录式分支"结构，每个版本都是一个独立的目录，可以直接查看和运行。

## 项目结构

```
step-by-step-agent/
├── README.md                          # 项目介绍、版本索引
├── CLAUDE.md                          # 项目设计文档（本文件）
├── docs/                              # 设计文档目录
│   ├── 001-minimal-agent/            # v001 设计文档
│   │   └── DESIGN.md
│   └── _template/                     # 设计文档模板
│       └── DESIGN.md
├── 001-minimal-agent/                 # v001: 最简 Agent
│   ├── agent/
│   │   ├── agent.py                   # Agent 用户接口
│   │   ├── loop.py                    # 主循环
│   │   ├── types.py                   # 数据类型定义
│   │   └── tools/                     # 工具实现
│   ├── run.py                         # 交互式运行入口
│   └── README.md
├── 002-xxx-xxx/                   # v002: xxx（待开发）
└── ...
```

## 设计理念

### 1. 目录式分支

**不使用 git 分支**，而是用目录管理不同版本：

- 每个版本一个独立目录 `001-xxx/`、`002-xxx/`
- 可以同时查看多个版本的代码
- 可以直接 `diff -r` 对比两个版本
- 降低学习门槛，不需要 git 操作

### 2. 设计文档驱动

每个版本必须包含设计文档 `docs/{version}/DESIGN.md`，回答：

1. **阶段目标**：这个版本解决什么问题、达到什么目标？
2. **设计思路**：做了什么设计？为什么这样设计？
3. **效果验证**：设计如何解决问题？
4. **遗留问题**：哪些问题留给后续版本？

### 3. 分层架构

v001 确立的核心架构：

```
┌─────────────────────────────────────┐
│   Agent (用户友好接口)               │
├─────────────────────────────────────┤
│   Loop (主循环: Think → Act → Observe) │
├─────────────────────────────────────┤
│   Types (数据模型定义)               │
├─────────────────────────────────────┤
│   Tools (工具实现)                   │
└─────────────────────────────────────┘
```

关键设计决策：
- **State vs Config 分离**：可变状态 vs 不可变配置
- **事件系统**：观察者模式解耦 Loop 和 UI
- **流式输出**：提升用户体验
- **并行工具执行**：提升效率

## 开发规范

### 代码规范
- **所有注释使用中文**：包括行内注释、文档字符串、函数说明等
- **变量和函数名使用英文**：保持代码的可读性和通用性
- **文档字符串使用中文**：模块、类、函数的文档说明使用中文

### 版本开发流程

1. **创建新版本目录**：复制上一个版本目录，命名为 `002-xxx/`
2. **创建设计文档**：复制 `docs/_template/DESIGN.md` 到 `docs/002-xxx/DESIGN.md`
3. **填写设计文档**：先写设计，再写代码
4. **实现代码**：按设计文档实现
5. **更新根目录 README**：添加新版本索引

### 设计文档书写规范

每个 `docs/{version}/DESIGN.md` 必须包含：

- **阶段目标**：清晰说明要解决的问题和要达到的目标
- **设计思路**：分点列出关键设计，每个设计都要解释"为什么这样设计"
- **效果验证**：用表格展示设计如何解决问题
- **未解决问题**：明确列出留给后续版本的问题
