# v001 Minimal Agent 测试方案

## 测试目标

针对 v001 的 4 个核心工具（read、write、edit、bash），验证 Agent 的：
- **规划能力**：能否合理拆解多步任务
- **工具选择倾向**：是否优先使用精准工具（如 edit 而非重写整个文件）
- **错误恢复能力**：edit 工具的唯一匹配限制下，失败后如何恢复
- **多步逻辑推理**：跨步骤的上下文记忆和逻辑连贯性

## 测试环境

- **模型**: deepseek-v4-pro
- **工作目录**: `001-minimal-agent/workspace/`
- **可用工具**: read、write、edit、bash

## 测试任务一：迭代式开发与自我验证

### 测试重点

文件创建、脚本执行验证、局部修改（观察是否使用 edit 而非重写整个文件）、处理报错的能力。

直接复制下面的 Agent Prompt，粘贴到 `python run.py` 的输入框里：

### Agent Prompt

直接复制下面这段，粘贴到 `python run.py` 的输入框里：
> 请帮我在当前工作目录下完成以下连贯的任务：
> 1. 创建一个名为 `calculator.py` 的 Python 脚本，里面包含一个计算列表平均值的函数 `calculate_average(numbers)`，并在脚本底部添加一段测试代码，传入 `[10, 20, 30, 40]` 并打印结果。
> 2. 运行这个脚本，确保它没有语法错误且能正确输出结果 25.0。
> 3. 在确认运行成功后，请修改 `calculator.py`：在打印平均值之前，额外添加一行代码，打印出"当前计算的列表长度是：X"（X需替换为实际长度）。
> 4. 再次运行脚本，确保修改成功且输出符合预期。

### 验证方法

1. **结果验证**：查看 workspace 目录下是否有 `calculator.py`，手动运行 `python calculator.py` 看看输出是否包含长度和平均值。
2. **Session 记录分析点**：
   - 第 3 步修改文件时，是选择直接覆盖整个文件，还是使用了精确替换？（优秀的 Agent 会为了节省 Token 尝试局部替换）
   - 如果使用了局部替换，是否一次性命中"唯一且完全匹配"的限制？如果替换失败（比如缩进不对或匹配到多处），观察它是如何恢复的（是重新读取文件确认内容，还是盲目重试？）

### 预期输出

```
当前计算的列表长度是：4
25.0
```

## 测试任务二：环境探索、数据处理与文本排版

### 测试重点

系统命令的灵活运用、多文件交互、长文本处理，以及在没有现成测试数据时自主构造数据的能力。

直接复制下面的 Agent Prompt，粘贴到 `python run.py` 的输入框里：

### Agent Prompt

> 我需要你帮我生成一份目录分析报告，请按以下步骤操作：
> 1. 生成 3 个包含随机英文单词的文本文件，分别命名为 `data_A.txt`, `data_B.txt`, `data_C.txt`（每个文件至少包含 5 行内容）。
> 2. 编写一个 Shell 脚本 `analyze.sh`，该脚本的作用是：找出当前目录下所有 `.txt` 文件，统计它们的总行数，并将结果（包括哪些文件以及总行数）输出到一个新文件 `report.log` 中。
> 3. 赋予脚本执行权限并运行它。
> 4. 查看生成的 `report.log` 文件。
> 5. 修改 `report.log` 文件，在其内容的**最首行**加上标题：">>> 自动生成的数据报告 <<<"。

### 验证方法

1. **结果验证**：检查 workspace 是否存在 3 个 data 文件、1 个可执行的 `analyze.sh`，以及最终的 `report.log`（首行必须有规定的标题，下方有统计结果）。
2. **Session 记录分析点**：
   - **命令构造能力**：在创建文件和跑 Shell 脚本时，它是否知道需要用 `chmod +x` 或者直接用 `bash analyze.sh` 来绕过权限问题？
   - **上下文记忆与读取**：在第 5 步修改 `report.log` 时，它是通过终端命令（如 `sed`）去修改，还是读取了文件再用文本编辑功能？
   - **边界处理**：在首行插入文本是一个经典的"精准替换"难题。因为"旧字符串"很难选定。优秀的 Agent 会先读取 `report.log` 的第一行内容，然后将其作为"旧字符串"，替换为"新标题 + \n + 原第一行"。重点观察它在这里的思维过程。

### 预期输出

`report.log` 首行必须是 `>>> 自动生成的数据报告 <<<`，下方包含各文件统计和总行数。
